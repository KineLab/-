# Структура Elixir/Phoenix проекта: Важные файлы

## 1. Ядро проекта (Elixir)

Это файлы, которые есть в любом Elixir-проекте, даже если он не связан с вебом.

```
/
├── mix.exs
├── mix.lock
├── config/
│   ├── config.exs
│   ├── dev.exs
│   └── prod.exs
├── lib/
│   └── kinelab_elixir/
│       └── application.ex
└── test/
```

*   `mix.exs`: **Паспорт и список покупок проекта.**
    *   Здесь мы указываем имя проекта, версию Elixir и, самое главное, список всех зависимостей (библиотек), которые нам нужны.

*   `mix.lock`: **"Замороженный" список покупок.**
    *   Этот файл гарантирует, что все разработчики в команде используют абсолютно те же версии библиотек, что и ты. `mix` создает и обновляет его автоматически.

*   `config/`: **Папка с настройками.**
    *   `config.exs`: **Главный пульт управления.** Основные настройки, которые применяются всегда.
    *   `dev.exs`: Настройки только для режима разработки (твоего компьютера).
    *   `prod.exs`: Настройки для "боевого" сервера, когда приложение увидят реальные пользователи.

*   `lib/`: **Мозг приложения.**
    *   Здесь живет вся основная логика твоего приложения, не связанная напрямую с вебом.
    *   `kinelab_elixir/application.ex`: **Главный дирижер.** Этот файл запускает и контролирует все основные процессы твоего приложения (например, базу данных, веб-сервер).

*   `test/`: **Отдел контроля качества.**
    *   Здесь лежат все тесты, которые проверяют, что твой код работает правильно.

## 2. Веб-часть (Phoenix)

Это файлы, которые добавляет фреймворк Phoenix для создания веб-интерфейса.

```
/
├── lib/
│   ├── kinelab_elixir/
│   │   └── repo.ex
│   └── kinelab_elixir_web/
│       ├── endpoint.ex
│       ├── router.ex
│       ├── components/
│       └── live/
├── priv/
│   └── repo/
│       └── migrations/
└── assets/
```

*   `lib/kinelab_elixir_web/`: **Лицо приложения.**
    *   В этой папке находится всё, что связано с вебом: страницы, компоненты, роутер.

*   `lib/kinelab_elixir_web/endpoint.ex`: **Главные ворота.**
    *   Каждая веб-запрос от браузера сначала попадает сюда. Этот файл — точка входа для всего веба.

*   `lib/kinelab_elixir_web/router.ex`: **Администратор / План дома.**
    *   Мы с ним уже работали. Он решает, какой код должен ответить на конкретный URL.

*   `lib/kinelab_elixir_web/live/`: **Комнаты для "живых" страниц (LiveView).**
    *   Здесь мы будем создавать наших "менеджеров" страниц, которые обновляются в реальном времени. Например, наш `welcome_live.ex`.

*   `lib/kinelab_elixir_web/components/`: **Набор деталей LEGO.**
    *   Здесь лежат переиспользуемые кусочки HTML и логики (кнопки, меню, карточки), которые можно вставлять в разные страницы.

*   `lib/kinelab_elixir/repo.ex`: **Наш личный библиотекарь.**
    *   Это модуль для общения с базой данных. Все запросы (найти, создать, обновить пользователя) идут через него.

*   `priv/repo/migrations/`: **Журнал изменений базы данных.**
    *   Здесь хранятся "миграции" — файлы, которые описывают, как создавать и изменять таблицы в твоей базе данных.

*   `assets/`: **Гардероб и косметика.**
    *   Папка для всего фронтенда: JavaScript (`app.js`), CSS-стили (`app.css`), картинки и т.д.
